## 1. 電子電路到底在幹嘛

先不要急著背元件名字。大部分電子系統，本質上都在做四件事：

1. 供電：把能量送到整個系統。
2. 感測：把外界狀態變成電訊號。
3. 判斷：由 MCU 或邏輯電路決定接下來要做什麼。
4. 執行：讓燈亮、馬達轉、蜂鳴器叫、繼電器切換。

用自走車當例子最直觀：

- 電池：供電。
- MCU：決定前進、後退、轉彎。
- 超音波感測器：量前方距離。
- 馬達與驅動器：讓輪子動起來。
- 電阻、電容、二極體：讓整套系統比較穩、比較不容易壞。

所以元件不是在互相競爭，而是在分工。初學時最有用的思考方式不是「這顆叫什麼」，而是「這顆在這個位置扮演什麼角色」。

你也可以把電路拆成兩條線去看：

- 能量流：電源怎麼送到負載。
- 訊號流：控制命令怎麼從 MCU 傳到其他元件。

例如用 MCU 控風扇：

- 能量流：`12V -> 風扇 -> MOSFET -> GND`
- 訊號流：`GPIO -> MOSFET Gate`

只要你習慣把這兩條線分開看，電路會清楚很多。

## 2. 先懂這些基本觀念，再學元件會快很多

### 2.1 電壓是什麼

白話講，電壓像「推力差」。不是完整物理定義，但很適合入門。  
常見電壓：

- 1.5V：乾電池
- 3.3V：很多 MCU 系統
- 5V：USB、部分模組
- 12V：風扇、馬達、常見變壓器
- 24V：工業控制常見

### 2.2 電流是什麼

白話講，電流是「實際流過去的量」。  
很多新手只看電壓，這不夠。你還要看電源能不能提供足夠電流，負載會不會瞬間拉很大電流。

### 2.3 電阻是什麼

電阻是對電流流動的阻礙。阻值越大，在相同條件下電流通常越小。

### 2.4 功率是什麼

功率代表每單位時間消耗或轉換多少能量。最常用公式：

```text
P = V x I
```

這個觀念會影響：

- 電阻會不會燙
- 穩壓器會不會過熱
- 電源夠不夠

### 2.5 GND 是什麼

GND 不只是負極，也常是整個系統的共同參考點。很多訊號的高低，都是相對於 GND 來定義。

### 2.6 開路與短路

- 開路：路不通，幾乎沒有電流流。
- 短路：不該直接相連的兩點被超低阻抗硬接在一起，通常很危險。

### 2.7 串聯與並聯

- 串聯：一個接一個排成一路。
- 並聯：兩端接在相同節點上。

這兩個概念會一直出現，像分壓、濾波、供電都會用到。

## 4. 電池與電源

### 4.0 先抓三件事

- 先看電壓對不對，再看電流能力夠不夠。
- 電源負極和系統 GND 通常要接在一起。
- 板子上的 `VIN`、`5V`、`3V3` 不能用猜的。

### 4.1 電池是什麼

電池就是能量來源。沒有穩定供電，再好的程式也沒用。

### 4.2 電池最重要看什麼

通常先看三件事：

- 電壓：能提供多大的推力差。
- 容量：大概能用多久。
- 可輸出電流能力：負載一大時撐不撐得住。

### 4.3 為什麼只看電壓不夠

例如一顆電源標 5V，但如果一接馬達就掉壓，系統還是可能重開機或不穩。

### 4.4 常見供電來源

- USB 5V
- 乾電池盒
- 鋰電池
- 實驗室電源
- 12V 變壓器
- 車用電瓶

### 4.5 電池最基本怎麼接

- 正極接系統的輸入電源端
- 負極接 GND

但真正重要的是：你要搞清楚板子上的 `VIN`、`5V`、`3V3`、`VCC` 各自代表什麼。

### 4.6 `VIN`、`5V`、`3V3` 為什麼不能混著想

不同板子標法不同：

- 有些 `VIN` 是外部輸入
- 有些 `5V` 是穩壓後節點
- `3V3` 常是 MCU 直接使用的供電

新手常見致命錯誤是把高電壓直接灌進 `3V3` 腳。

### 4.7 為什麼共地很重要

通常下面這些都要共地：

- 電池 GND
- MCU GND
- 驅動器 GND
- 感測器 GND

不共地時，訊號可能沒有共同參考點，導致：

- UART 不通
- ADC 亂跳
- MOSFET 控制異常

### 4.8 電池可以直接接 MCU 嗎

看電壓範圍。  
3.3V MCU 不能直接吃 12V。  
有些板子可從 `VIN` 進較高電壓，再由板上穩壓器處理；有些則不行，必須先查板子說明。

### 4.9 掉壓是什麼感覺

常見症狀：

- 馬達一啟動，MCU 重開機
- 模組一發射，系統亂掉
- LED 明顯變暗
- 通訊突然不穩

很多這類問題，本質都是供電扛不住。

### 4.10 電源這章最該記住的事

- 電壓對，不代表電源就夠用。
- 還要看電流能力。
- 系統常常需要共地。
- `VIN`、`5V`、`3V3` 不一定能互相替代。
- 系統怪怪的時候，先查供電很常是對的。

## 5. 電阻

### 5.0 先抓三件事

- 電阻最常做三件事：限流、分壓、拉高拉低。
- 電阻有壓降的前提，是有電流流過。
- 算完阻值後，還要回頭看功率和是否有標準值可選。

### 5.1 電阻是什麼

電阻最白話的理解，不是「把電吃掉」，而是「限制電流流動，並在有電流流過時形成電壓降」。

你可以把它想成水管裡面的阻力：

- 阻力越大，水越不容易流
- 電阻越大，電流越不容易流

但這個比喻只能幫你入門。真正做設計時，你更常用的是這三件事：

- 限制電流
- 分配電壓
- 定義訊號狀態

### 5.2 電阻通常有方向嗎

一般固定電阻通常沒有方向。  
正著接、反著接，阻值都一樣。

例外是某些特殊元件，例如：

- 熱敏電阻
- 光敏電阻
- 可變電阻
- 電阻網路模組

但你現在先把「一般電阻沒有方向」記起來就夠了。

### 5.3 電阻會不會讓電壓下降

會，但要把條件說完整。

正確說法是：

> 當有電流流過電阻時，電阻兩端就會出現電壓降。

公式是：

```text
V = I x R
```

這句話對新手非常重要，因為很多人會誤會成：

- 只要有電阻，電壓就一定會變低
- 電阻天生會把電壓吃掉

其實不是。要有電流流過，才會有壓降。

### 5.4 如果沒有電流，還會有壓降嗎

理想情況下，不會。

例如你拿一顆電阻串在電路裡，但後面根本沒有接成回路，沒有電流流動，那這顆電阻上就不會出現你期待的壓降。

所以工程上常講一句很重要的話：

> 電壓降不是電阻自己憑空產生的，而是電流流過去之後出現的結果。

### 5.5 最常見用途一：LED 限流

最經典的入門電路：

```text
GPIO -> 電阻 -> LED -> GND
```

LED 不是會自己限制電流的元件。  
如果你直接把 LED 接到電源上，電流可能衝得很大，最後 LED 很快就壞掉。

這時電阻的工作就是：

- 把多出來的電壓分擔掉
- 把流過 LED 的電流限制在安全範圍

### 5.6 為什麼 LED 常用 220Ω、330Ω、1kΩ

這不是魔法數字，而是因為它們剛好落在常見、好買、也常常夠用的範圍。

以 3.3V 系統、紅色 LED、目標電流 3mA 到 8mA 為例：

- 紅 LED 順向電壓常約 1.8V 到 2.2V
- 剩下要落在電阻上的電壓常約 1V 多一點

例如估算：

```text
3.3V - 2.0V = 1.3V
```

如果希望：

- 6mA，電阻約 `1.3 / 0.006 = 216Ω`
- 4mA，電阻約 `1.3 / 0.004 = 325Ω`
- 1.3mA，電阻約 `1.3 / 0.0013 = 1kΩ`

所以：

- `220Ω` 亮度通常較明顯
- `330Ω` 是很常見的穩健起手值
- `1kΩ` 比較省電、比較不刺眼，也對 GPIO 更溫和

### 5.7 最常見用途二：分壓

兩顆電阻串聯，可以把一個電壓分成你要的比例。

```text
VCC -> R1 -> 中點 -> R2 -> GND
```

中點電壓：

```text
Vout = Vin x R2 / (R1 + R2)
```

這在很多地方都很常見：

- 電池電壓量測
- 把 5V 降到 ADC 能接受的範圍
- 類比感測器偏壓

### 5.8 分壓為什麼不能亂選阻值

因為分壓不是只有「比例對就好」。

還要看：

- 分壓器本身浪不浪費電
- 後面接的電路會不會把中點拉歪
- ADC 取樣時，訊號源阻抗會不會太高

如果阻值選太小：

- 很耗電
- 白白讓電阻一直發熱

如果阻值選太大：

- 雜訊影響比較明顯
- ADC 讀值可能不穩
- 容易被後級輸入電流影響

所以工程上常常不是只算比例，還要一起看：

- 功耗
- 精度
- ADC 規格
- 輸入阻抗

### 5.9 最常見用途三：上拉 / 下拉

按鍵或數位輸入如果沒有被明確拉到高或低，就可能浮動。

這時會用一顆阻值比較大的電阻，輕輕把它拉到固定狀態：

- 上拉：拉向 VCC
- 下拉：拉向 GND

這裡阻值為什麼常是 `4.7kΩ`、`10kΩ`、`47kΩ`？

因為這是一種折衷：

- 太小：浪費電
- 太大：抗干擾變差，狀態容易飄

很多一般數位輸入場合，`10kΩ` 是很好的入門起手值。

### 5.10 I2C 上拉為什麼又常見 2.2kΩ、4.7kΩ

因為 I2C 不是普通按鍵輸入，它的線路要靠上拉電阻把訊號拉回高電位。

阻值太大時：

- 上升太慢
- 高速通訊可能失敗

阻值太小時：

- 裝置拉低時要承受比較大電流
- 功耗增加

所以 I2C 上拉值要看：

- 匯流排速度
- 線長
- 總電容
- 裝置數量

`4.7kΩ` 常見，因為在很多短線、一般速度場合都能工作。

### 5.11 電阻值為什麼都是這些奇怪數字

你會常看到：

- 220Ω
- 330Ω
- 470Ω
- 1kΩ
- 4.7kΩ
- 10kΩ

這不是隨便訂的，而是來自標準阻值系列，例如：

- E6
- E12
- E24

廠商不可能生產所有連續數字的阻值，所以會用一組標準化數列，讓不同誤差等級下的阻值彼此平均分布。

所以你算出 `260Ω`，實際常會改拿：

- `270Ω`
- `240Ω`
- `330Ω`

這是正常工程做法。

### 5.12 電阻也有功率額定

不是只看阻值。  
電阻如果吃太多功率會過熱，甚至燒焦。

功率可用這些公式估：

```text
P = V x I
P = I^2 x R
P = V^2 / R
```

例如一顆 `1kΩ` 電阻上掉 `12V`：

```text
P = 12^2 / 1000 = 0.144W
```

這時：

- `1/8W` 電阻可能已經很勉強
- `1/4W` 會比較安全

### 5.13 實際工程中怎麼選電阻

入門時可以用這個順序想：

1. 先決定它的用途
2. 再算需要的阻值範圍
3. 再選最接近的標準值
4. 再檢查功率和精度夠不夠

例如：

- LED 限流重點是電流和亮度
- 分壓重點是比例、功耗、ADC 讀值穩不穩
- 上拉重點是穩定度和功耗平衡
- 電流感測重點是低阻值、高精度、功率

### 5.14 電阻最常見錯誤

- LED 沒串電阻
- 以為電阻一定會固定掉某個電壓
- 分壓只算比例，沒看後級負載
- 只看阻值，不看功率
- 上拉 / 下拉太弱，導致訊號容易飄

### 5.15 電阻這章一句話總結

電阻不是單純「擋電流」而已，它是拿來控制電流、安排電壓、定義訊號狀態的基礎工具。

## 6. 電容

### 6.0 先抓三件事

- 小電容常顧高頻和去耦，大電容常顧供電緩衝。
- 陶瓷電容常放在 IC 旁邊，電解電容常放在電源入口或大負載附近。
- 去耦電容離 IC 太遠，效果通常會變差。

### 6.1 電容是什麼

白話講，電容像一個暫時能存放電荷的小緩衝區。  
但在實務裡，電容更常扮演的是：

- 電源穩定器
- 雜訊吸收器
- 暫時能量緩衝器
- 訊號耦合元件
- 濾波元件

### 6.2 電容最常見功能

最值得初學者先熟的是這五種：

1. 去耦
2. 電源平滑
3. RC 定時
4. AC 耦合
5. 濾波

### 6.3 去耦是什麼

MCU、邏輯 IC、驅動器在切換時，常會突然需要一小口瞬間電流。  
如果這個電流每次都要從很遠的電源線拉過來，線路的寄生電感和阻抗會讓局部電壓晃動。

去耦電容放在 IC 電源腳旁邊，就是要做這件事：

> 在很近的地方先放一小桶電，當 IC 突然要喝電時，先由它補上。

### 6.4 為什麼去耦電容要靠近 IC

因為實際電路不是理想導線。

走線越長，通常就意味著：

- 額外電阻
- 額外寄生電感
- 額外雜訊耦合機會

所以去耦電容如果離電源腳太遠，原本應該很快補上的電流，可能就慢了一拍，效果變差很多。

### 6.5 常見電容值直覺

下面這些不是死規則，但很常見：

- `0.1uF`：數位 IC、MCU 去耦基本款
- `1uF`：局部穩定、部分模組輔助去耦
- `10uF`：電源入口、小負載變動緩衝
- `47uF`、`100uF` 以上：馬達、驅動器、電源輸入、瞬間大電流緩衝

很多板子不是只放一顆，而是：

- 一顆小的處理快變化
- 一顆大的處理慢一點但能量比較大的變化

### 6.6 陶瓷電容與電解電容怎麼選

這是你提到很重要的一點，實作上真的不能只寫一句話帶過。

先講最白話的差異：

- 陶瓷電容：小、快、常拿來對付高頻與去耦
- 電解電容：容量大、便宜、常拿來做電源儲能和平滑

### 6.7 陶瓷電容的特性

優點：

- 通常沒有極性
- ESR 通常較低
- 對高頻雜訊很有效
- 體積小
- 很適合貼在 IC 旁邊

缺點：

- 大容量時價格和體積可能上升
- 某些材質在偏壓下有效電容會下降
- 太大顆時不一定是最划算的能量緩衝方案

最典型用途：

- MCU 去耦
- 邏輯 IC 旁路
- 高頻濾波
- 類比前端局部穩定

### 6.8 電解電容的特性

優點：

- 很容易做到較大容量
- 價格通常划算
- 適合當電源入口或大負載附近的能量緩衝

缺點：

- 通常有極性
- ESR 較高
- 對高頻雜訊通常不如小陶瓷電容靈活
- 壽命與溫度、紋波電流有關

最典型用途：

- 電源輸入平滑
- 馬達驅動附近
- 繼電器 / 電磁閥電源緩衝
- 穩壓器輸入或輸出端的大容量儲能

### 6.9 為什麼板子上常常兩種都放

因為它們擅長的頻段和任務不同。

很常見的搭配是：

```text
電源入口：10uF ~ 100uF 電解或固態電容
IC 腳邊：0.1uF 陶瓷電容
```

白話理解：

- 大電容像大水桶，能補比較久
- 小陶瓷像離你嘴巴很近的小水杯，拿來快速應付瞬間需求

### 6.10 陶瓷電容實際怎麼選

實作上你會看：

- 容值
- 額定電壓
- 封裝大小
- 材質類型，例如 C0G、X7R、Y5V

新手先記這個版本就很夠用：

- 一般去耦優先考慮 `X7R`
- 精密、溫漂要求高才看 `C0G/NP0`
- 不要太依賴 `Y5V` 這種特性較差的材質做關鍵容量

### 6.11 電解電容實際怎麼選

你要看：

- 容值夠不夠
- 額定電壓要留裕量
- ESR
- 溫度等級
- 壽命
- 體積

例如 12V 系統不建議只拿 10V 電解去用。  
通常會留裕量，像是選：

- 16V
- 25V

這會比把規格壓得很死安全。

### 6.12 電容額定電壓怎麼看

電容不是只有容量，額定電壓同樣重要。

如果你在 5V 電路裡放一顆 6.3V 陶瓷電容，有時可能可用，但裕量偏少。  
如果環境有突波、溫度變化，工程上常會留更舒服的空間。

入門可先養成這種習慣：

- 3.3V 系統常看 6.3V 或 10V
- 5V 系統常看 10V 或 16V
- 12V 系統常看 16V 或 25V

### 6.13 電容在馬達和驅動器附近為什麼重要

因為馬達與驅動器常有：

- 啟動大電流
- PWM 切換
- 瞬間電流拉扯

這些都會讓電源線上的電壓出現波動。  
如果附近沒有足夠的電容，可能會發生：

- MCU 重開機
- 感測器讀值亂跳
- 通訊失敗
- 驅動器誤動作

### 6.14 實際工程中怎麼選電容

可以先用這種思路：

1. 先搞清楚用途
2. 再決定要高頻去耦還是大能量緩衝
3. 再選電容類型
4. 再看額定電壓和安裝位置

如果是：

- IC 腳邊去耦：優先小陶瓷
- 電源入口：常加大容量電容
- 馬達驅動附近：大容量 + 小陶瓷一起上
- 音訊耦合：會看容值、漏電、極性、失真需求

### 6.15 電容最常見錯誤

- 電解電容接反
- 只看容量，不看額定電壓
- 以為一顆大電容可以取代所有小電容
- 去耦電容離 IC 太遠
- 忽略陶瓷電容在偏壓下有效電容會變小

### 6.16 電容這章一句話總結

電容不是只有「存電」而已，在實際工程裡，它非常常負責把電源和訊號從「會亂」拉回「能穩定工作」。

## 7. 電感

### 7.0 先抓三件事

- 電感最常出現在 Buck、Boost、濾波和線圈類元件附近。
- 你第一次看不懂沒關係，先知道它在管理電流變化。
- 只要看到開關電源，幾乎就該想到它。

### 7.1 電感是什麼

白話講，電感討厭電流突然改變。

如果你硬要讓流過它的電流一下子變大或變小，它就會用產生電壓的方式反抗這個變化。

這也是為什麼電感在：

- DC-DC 轉換器
- 線圈
- 馬達
- 繼電器
- 電磁閥

這些世界裡那麼重要。

### 7.2 電感最常出現在哪

- Buck 降壓電源
- Boost 升壓電源
- EMI 濾波
- 馬達與驅動器周邊
- 共模濾波器

### 7.3 為什麼它常跟 DC-DC 一起出現

因為開關電源的高效率，不是靠把多餘能量變熱消耗掉，而是靠：

- 先暫時存進電感
- 再在適當時間釋放給負載

這和 LDO 完全是不同邏輯。

### 7.4 電感與電容常常一起合作

很常看到：

- 電感負責處理電流變化
- 電容負責平滑電壓波動

所以很多電源濾波、DC-DC 輸出，都會看到 L 和 C 成對出現。

### 7.5 初學者要先記住的重點

- 電感不是高級版電阻
- 電感不是拿大容量電容就能替代
- 只要碰到開關電源和感性負載，通常都會遇到它

### 7.6 電感這章一句話總結

你前期不用把公式全部背熟，但一定要知道：電感是能量轉移和電流變化管理的主角。

## 8. 二極體

### 8.0 先抓三件事

- 二極體和 LED 一樣，有方向，不能像電阻那樣隨便反接。
- 順向電壓和逆向耐壓都很重要。
- 線圈負載旁邊的保護二極體，很多時候不是可有可無。

### 8.1 二極體是什麼

最白話版本：它比較容易讓電流往單一方向通過。

這句話雖然不夠精確，但對入門非常實用。  
先用它建立方向感，再慢慢補細節。

### 8.2 二極體最常見用途

- 防反接
- 整流
- Flyback 保護
- 鉗位保護
- 穩壓參考

### 8.3 二極體有方向嗎

有，而且非常重要。

一般你會看到：

- Anode，陽極
- Cathode，陰極

封裝上常有一條線，通常是陰極標記。

### 8.4 什麼是順向電壓

這是你特別點名要補的重點。

當二極體以「容易導通的方向」接上去時，電流不是一到就毫無阻礙地流，而是通常會有一個典型壓降，這就是順向電壓。

常見直覺值：

- 一般矽二極體：約 `0.6V` 到 `0.7V`
- Schottky：常約 `0.2V` 到 `0.4V`
- LED：依顏色和材料不同，常約 `1.8V` 到 `3V` 以上

所以你會常聽到：

> 矽二極體導通會掉 0.7V 左右

這句不是宇宙真理，但對初學很有用。

### 8.5 順向電壓代表什麼

代表當電流經過這顆二極體時，有一部分電壓會落在它身上。

例如你在 5V 供電路徑上串一顆普通二極體做防反接：

- 進來 5V
- 經過二極體後可能只剩 4.3V 左右

所以設計上不能只想「有保護真好」，還要問：

- 這個壓降我能不能接受
- 後面電路會不會因此吃不到足夠電壓

### 8.6 什麼是逆向電壓

就是你把二極體反著接，讓它進入不想導通的方向。

理想狀況下，二極體逆向時應該完全不導通。  
但實際上：

- 會有很小漏電流
- 如果逆向電壓太大，可能擊穿

### 8.7 逆向耐壓為什麼重要

每顆二極體都有限定它最多能承受多少逆向電壓。

如果超過：

- 輕則漏電變大
- 重則擊穿損壞

所以選二極體時不能只看「導不導得通」，還要看：

- 順向電流夠不夠
- 順向壓降能不能接受
- 逆向耐壓夠不夠

### 8.8 為什麼感性負載關閉時會冒出反向高壓尖峰

這是你提到的核心問題，這裡要講完整。

像這些元件都屬於感性負載：

- 繼電器線圈
- 電磁閥
- 馬達繞組
- 接觸器線圈

它們的共同點是：裡面有線圈，線圈本質上就有電感。

而電感最討厭的事情就是：

> 流過它的電流被突然硬切掉

當線圈通電時，裡面存了一部分磁場能量。  
你把開關突然關掉時，電流原本還想繼續流，但路被你切斷了。

這時線圈會做什麼？

它會自己產生一個電壓，想盡辦法把原本的電流繼續推出去。

如果沒有安全的路徑讓這股電流走，結果就會變成：

- 開關兩端出現很高的反向電壓
- 晶體管承受突波
- MOSFET 或 BJT 被打壞
- 繼電器觸點冒火花
- MCU 被雜訊干擾甚至重開機

### 8.9 為什麼會是高壓

因為電感的本性就是：

> 你越想讓電流一下子改變，它就越用更大的電壓來對抗。

如果切斷得很快，而又沒有釋放路徑，這個電壓可以高到遠超過原本供電電壓。

所以不是說：

- 你原本用 12V
- 關掉最多也就 12V

不是。  
在突波瞬間，可能遠高於 12V。

### 8.10 Flyback diode 到底怎麼保護

在線圈兩端並聯一顆方向正確的二極體。

平常通電時：

- 這顆二極體是反向偏壓
- 幾乎不導通

關閉瞬間：

- 線圈想維持原本電流方向
- 電壓極性翻轉
- 二極體轉成順向導通
- 線圈能量改在線圈和二極體的回路裡慢慢釋放

結果就是：

- 電壓尖峰被壓住
- 晶體管比較安全
- 雜訊小很多

### 8.11 Flyback diode 的代價是什麼

不是沒有代價。

因為它讓線圈電流比較慢地衰減，所以有些元件的釋放速度會變慢，例如：

- 繼電器斷開速度稍慢
- 電磁閥回位速度稍慢

如果系統很在意釋放速度，可能會改用：

- TVS
- Zener + diode
- RC snubber

這些做法會是更進階的保護設計。

### 8.12 二極體家族有哪些

你會遇到：

- 一般整流二極體
- 快恢復二極體
- Schottky 二極體
- Zener 二極體
- TVS 二極體
- LED

它們不是誰高級誰低級，而是用途不同。

### 8.13 實際工程中怎麼選二極體

你至少要看這幾件事：

- 順向電流額定
- 逆向耐壓
- 順向壓降
- 恢復速度
- 封裝和散熱

例如：

- 電源防反接，會在意壓降
- Flyback，會在意耐流和耐壓
- 開關電源，會在意速度和損耗

### 8.14 二極體最常見錯誤

- 接反
- 只看名字，不看規格
- 線圈負載沒做 flyback 保護
- 以為所有二極體都能互相替代

### 8.15 二極體這章一句話總結

二極體最重要的不只是「單向導通」，而是你要知道它能拿來做方向控制、保護、整流與突波管理。

## 9. LED

### 9.0 先抓三件事

- LED 有方向。
- LED 幾乎都要限流。
- 板載 LED 很常是 active-low，不一定是 GPIO 拉高才亮。

### 9.1 LED 是什麼

LED 是會發光的二極體，所以它同時繼承了二極體最重要的兩個性格：

- 有方向
- 有順向壓降

另外它還有一個你一定要記住的實務重點：

- 幾乎都要限流

### 9.2 LED 的順向電壓大概是多少

不同顏色常不一樣，粗略感覺可先記：

- 紅色：約 `1.8V` 到 `2.2V`
- 黃色 / 綠色：約 `2V` 左右到 `2.4V`
- 白色 / 藍色：約 `2.8V` 到 `3.3V` 左右

這些只是常見範圍，不是固定值。

### 9.3 LED 最基本接法

```text
GPIO -> 電阻 -> LED -> GND
```

或

```text
VCC -> 電阻 -> LED -> GPIO
```

第二種常見在 active-low 控制。

### 9.4 為什麼 LED 不能直接接電源

因為 LED 不是線性負載。  
只要電壓一到某個程度，電流就可能快速升高。

如果沒有電阻或恆流驅動去限制它，可能會：

- 電流過大
- 接面發熱
- 亮一下就壞

### 9.5 LED 可以直接接 GPIO 嗎

小指示燈在低電流下通常可以，但要記得：

- 每腳最大電流有限
- 全部 GPIO 總電流也有限
- 不同 MCU 規格差很多

所以工程上如果 LED 很多、很亮、或要長時間點亮，常會：

- 增加驅動級
- 用 LED driver
- 降低單顆 LED 電流

### 9.6 怎麼選 LED 電阻

先用這個思路：

1. 先看供電電壓
2. 再估 LED 順向壓降
3. 決定要多亮
4. 算電阻值
5. 再選最接近的標準值

公式：

```text
R = (Vsupply - Vf) / I
```

### 9.7 LED 最常見錯誤

- 沒串電阻
- 接反
- 以為所有顏色的順向電壓一樣
- 只追求越亮越好，忽略 GPIO 電流能力

### 9.8 LED 這章一句話總結

LED 是最適合練基本功的元件，因為它會逼你同時理解極性、順向電壓、限流與 GPIO 驅動能力。

## 10. BJT

### 10.0 先抓三件事

- BJT 很常拿來當開關，也可以拿來當放大器。
- Base 前面幾乎都要有電阻。
- 小電流、便宜、簡單控制時，BJT 常常很好用。

### 10.1 BJT 是什麼

BJT 是雙極性電晶體。  
對 MCU 初學者來說，最實用的理解有兩個：

- 它可以當開關
- 它也可以當放大器

### 10.2 為什麼叫用小電流控制大電流

以 NPN 為例，Base 是控制端。  
當 Base 有適當電流流入時，Collector 到 Emitter 的主路徑就能導通，讓較大的電流流過負載。

所以它不像 MOSFET 主要是用電壓控制。  
BJT 比較像是：

> 你先給它一點控制電流，它再幫你放大出更大的主電流。

### 10.3 常見腳位

以 NPN 為例：

- Base：控制端
- Collector：主電流路輸入側
- Emitter：主電流路輸出側

PNP 的方向和邏輯會相反一些，但你先把 NPN 學熟最重要。

### 10.4 BJT 什麼時候當開關，什麼時候當放大器

這是很重要的觀念。

如果拿來當開關，你會希望它只有兩種狀態：

- 截止，不導通
- 飽和，盡量完全導通

如果拿來做放大器，它則工作在中間區域，讓輸入的小變化對應成輸出的較大變化。

對 MCU 控制負載來說，你通常先學「開關用法」。

### 10.5 最常見接法：NPN 低邊開關

```text
VCC -> 負載 -> Collector
Emitter -> GND
GPIO -> 電阻 -> Base
```

當 GPIO 輸出高，Base 有電流，BJT 導通，負載得電。

### 10.6 為什麼 Base 要串電阻

因為 Base-Emitter 之間不是你可以直接硬短進去的。

Base 電阻的目的：

- 限制基極電流
- 保護 MCU
- 讓設計落在可預期範圍

### 10.7 Base 電阻怎麼估

開關用途時，常先知道：

- 負載需要多少 Collector 電流
- MCU 可提供多少 GPIO 電流

再用保守方式選一個基極電流，確保 BJT 能進入飽和。

入門不一定要先硬背飽和設計公式，但要有這個直覺：

- 負載電流越大，Base 也通常要給得更有力
- 如果 Base 給太弱，BJT 可能進不了飽和，結果發熱增加

### 10.8 BJT 的缺點是什麼

如果拿來當開關，BJT 常見缺點是：

- 需要 Base 電流
- 導通時有壓降
- 大電流場合效率常不如 MOSFET

所以負載變大之後，很多人會改用 MOSFET。

### 10.9 BJT 的優點是什麼

它也不是落伍元件。

BJT 的優點常是：

- 便宜
- 好買
- 小電流驅動很方便
- 類比放大領域很常用
- 某些簡單驅動電路很好設計

### 10.10 實際工程中什麼時候選 BJT

常見場景：

- 小電流繼電器線圈驅動
- 有源蜂鳴器
- 小燈、小指示負載
- 簡單電平轉換
- 類比前端的小訊號放大

如果是：

- 電流更大
- 發熱更在意
- 要做 PWM

通常就會更偏向 MOSFET。

### 10.11 BJT、MOSFET、繼電器怎麼選

這是很實用的工程問題。

如果你要控制的是直流負載，而且：

- 電流不大
- 成本要低
- 電路要簡單

可以先考慮 BJT。

如果你要：

- 控較大電流
- 低損耗
- 高效率
- PWM 調速或調光

通常優先考慮 MOSFET。

如果你要：

- 切 AC
- 要電氣隔離
- 要像真正機械開關那樣切換

通常會考慮繼電器。

### 10.12 BJT 最常見錯誤

- Base 沒串電阻
- 腳位看錯
- 低估負載電流
- 線圈負載沒加 flyback diode
- 以為所有 BJT 腳位排列都一樣

### 10.13 BJT 這章一句話總結

BJT 是初學者一定要懂的控制元件，因為它會讓你真正進入「MCU 只是控制，真正驅動要靠外部元件」這個世界。

## 11. MOSFET

### 11.0 先抓三件事

- MOSFET 是 MCU 控大多數直流負載時最常見的開關元件。
- `Vgs(th)` 不是「完全打開電壓」，選型要看 `Rds(on)`。
- Gate 下拉和共地，都是新手超容易漏掉的關鍵。

### 11.1 MOSFET 是什麼

MOSFET 是非常常用的功率開關元件。  
在 MCU 控制負載的世界裡，它幾乎是主力。

### 11.2 它和 BJT 最大差別是什麼

很白話地說：

- BJT 比較像用電流控制
- MOSFET 比較像用電壓控制

這讓 MOSFET 在很多開關應用裡更省控制端負擔，也更容易做到低損耗。

### 11.3 常見腳位

- Gate：控制端
- Drain：主電流路一端
- Source：主電流路另一端

### 11.4 最常見接法：N-MOS 低邊開關

```text
VCC -> 負載 -> Drain
Source -> GND
GPIO -> 小電阻 -> Gate
Gate -> 下拉電阻 -> GND
```

這個架構超常見，因為：

- 容易理解
- 容易驅動
- 很適合 MCU 控制直流負載

### 11.5 為什麼 Gate 常有兩顆電阻

一顆串在 GPIO 和 Gate 之間，常見幾十歐姆到幾百歐姆。

用途：

- 限制切換瞬間電流
- 抑制振鈴
- 保護驅動端

另一顆是 Gate 到 GND 的下拉。

用途：

- 在 MCU 還沒開機時把 Gate 拉在安全狀態
- 避免 Gate 浮動導致負載亂開

### 11.6 為什麼共地超重要

MOSFET 不是看 Gate 對地幾伏，而是看：

```text
Vgs = Vgate - Vsource
```

如果 MCU 地和負載地沒有連在一起，Gate 的參考點就不對，MOSFET 可能：

- 打不開
- 開不完全
- 動作不穩

### 11.7 什麼是邏輯電平 MOSFET

這代表它在較低的 Gate 電壓下，例如：

- 2.5V
- 3.3V
- 4.5V

就能有不錯的導通能力。

### 11.8 為什麼不能只看 `Vgs(th)`

這是新手最常被誤導的地方。

`Vgs(th)` 的意思通常只是：

> MOSFET 開始有一點點導通的門檻附近

不是：

> 已經完全打開，可以拿來安心帶大電流

所以選型時真正該看的，是資料表上：

- 在 `Vgs = 2.5V`、`4.5V` 或 `10V` 時的 `Rds(on)`

如果資料表只保證 `10V` 時很低阻，但你系統只有 `3.3V` GPIO，那可能不適合。

### 11.9 `Rds(on)` 為什麼重要

因為 MOSFET 導通後不是完全沒有阻抗，而是像一個很小的電阻。

它越小：

- 壓降越小
- 發熱越低
- 效率越高

例如負載電流 3A，如果 `Rds(on)` 是 `0.1Ω`：

```text
P = I^2 x R = 3^2 x 0.1 = 0.9W
```

0.9W 對一顆小封裝 MOSFET 已經很可能偏熱。

### 11.10 MOSFET 為什麼很適合 PWM

因為它可以很快地開關，而且導通損耗通常比 BJT 小。  
所以控制：

- 風扇
- 馬達
- LED 燈條
- 加熱器

都很常用 MOSFET 做 PWM。

### 11.11 MOSFET 的 body diode 是什麼

MOSFET 內部通常天生帶有 body diode。  
這顆內建二極體在某些拓樸裡很重要，但也可能帶來你不想要的電流路徑。

所以不是只看「反正裡面有二極體」，而是要搞清楚：

- 電流可能從哪裡偷跑
- 這個拓樸是不是會被 body diode 影響

### 11.12 實際工程中什麼時候選 MOSFET

如果你要控制的是：

- 直流馬達
- 風扇
- LED 燈條
- 加熱片
- 電磁閥

而且希望：

- 損耗低
- 可以 PWM
- 電流比較大

MOSFET 通常是首選。

### 11.13 什麼時候不要只靠單顆 MOSFET

如果你要做的是：

- 正反轉控制
- 高邊切換
- 大電流同步整流
- 需要快速切換和保護

常常不會只用單顆 MOSFET，而是：

- H 橋驅動器
- Half-bridge driver
- 專用 high-side driver
- 馬達驅動 IC

### 11.14 MOSFET、BJT、繼電器怎麼選

可以先用這個超實用版本判斷：

- 要切 AC 或要隔離：先想繼電器
- 要切 DC、要 PWM、效率重要：先想 MOSFET
- 小負載、便宜、簡單：BJT 也很常夠用

### 11.15 MOSFET 最常見錯誤

- 只看 `Vgs(th)`，沒看 `Rds(on)`
- Drain / Source 接反
- 沒共地
- 沒有 Gate 下拉
- 感性負載沒做保護
- 忽略散熱

### 11.16 MOSFET 這章一句話總結

MOSFET 是實際工程裡最重要的開關元件之一，學會它，你才真正有能力安全又有效率地控制大多數直流負載。

## 12. 放大器 / 運算放大器

### 12.0 先抓三件事

- 這章第一次看不懂很正常，不用有壓力。
- 放大器不只是把訊號變大，也常拿來做緩衝和濾波。
- 你現在先記住「很多感測器訊號不能直接丟給 MCU」就很夠用了。

### 12.1 放大器是什麼

放大器最白話的意思是：

> 把原本太小、太弱、太容易被干擾的訊號，整理成比較好用的訊號。

很多新手以為放大器就是把數字乘大。  
其實在工程上，它常做的不只放大，還包括：

- 緩衝
- 濾波
- 偏移
- 比較前級
- 阻抗轉換
- 感測器訊號調理

### 12.2 為什麼 MCU 初學者也該認識它

因為很多感測器不是一接 ADC 就完美：

- 訊號太小
- 雜訊太多
- 輸出阻抗太高
- 中心點不對
- 波形不適合直接量

這時運放就是類比世界的整理工具。

### 12.3 運算放大器最基本怎麼理解

你先不要急著背完整理論。  
入門先記：

- 它有兩個輸入端
- 一個輸出端
- 常搭配負回授工作

然後用外部電阻和電容決定它的行為。

### 12.4 最常見功能一：非反相放大

把輸入訊號放大，而且輸出方向不反轉。  
很適合初學者理解「小訊號變大」這件事。

### 12.5 最常見功能二：反相放大

輸出會和輸入相反方向。  
雖然一開始看起來抽象，但它是很經典的基本電路。

### 12.6 最常見功能三：緩衝器

這個非常重要，卻很容易被低估。

有時你不是想把訊號放很大，而是：

- 不希望後級把前級拖垮
- 想讓 ADC 看到比較穩定的訊號源
- 想讓高阻抗感測器比較好帶後面電路

這時會用 voltage follower，也就是緩衝器。

### 12.7 最常見功能四：濾波

運放可以和電阻電容組合成各種濾波器，例如：

- 低通
- 高通
- 帶通

如果你感測器輸出很吵，先濾掉高頻雜訊再送 ADC，常常會穩很多。

### 12.8 運放和比較器一樣嗎

不完全一樣。

雖然有些情況下運放可以拿來做比較，但工程上：

- 運放比較擅長在線性區域處理訊號
- 比較器更擅長快速判斷高低

如果你要穩定做門檻判斷、數位化切換，很多時候比較器更合適。

### 12.9 運放實際上最常用在哪

- 電流感測放大
- 麥克風前級
- 壓力感測器訊號放大
- 熱電偶放大
- ADC 前端緩衝
- 主動濾波

### 12.10 選運放要看什麼

這裡是工程上很重要的基本盤：

- 供電電壓範圍
- 能不能單電源工作
- 輸入共模範圍
- 輸出能不能接近軌到軌
- 頻寬
- slew rate
- offset
- 噪聲

### 12.11 什麼是 rail-to-rail

如果你用 3.3V 單電源系統，很多時候你會希望：

- 輸入可以接近 0V 和 3.3V
- 輸出也可以盡量接近 0V 和 3.3V

這時 rail-to-rail input/output 的能力就很重要。

否則你可能會遇到：

- 明明供電 3.3V
- 但輸出拉不到接近 3.3V
- 或輸入接近 GND 時反而不準

### 12.12 為什麼很多感測器前面需要運放

因為 MCU ADC 通常喜歡的是：

- 一個在 0V 到 3.3V 之間
- 來源阻抗不要太高
- 雜訊不要太大
- 不要超過腳位保護範圍

但真實感測器輸出常常不是這樣。  
所以運放就負責把感測器翻譯成 ADC 好消化的形式。

### 12.13 運放最常見錯誤

- 以為所有運放都能單 3.3V 工作
- 忽略輸入共模範圍
- 忽略輸出擺幅限制
- 直接拿來當比較器用卻要求很快切換
- 沒處理好電源去耦

### 12.14 放大器這章一句話總結

放大器不是類比高手才需要學，它是你把真實世界訊號成功送進 MCU 的重要橋樑。

## 13. 穩壓器：LDO 與 DC-DC

### 13.0 先抓三件事

- 穩壓器的核心工作，就是把輸入電源整理成穩定輸出。
- LDO 簡單，但大壓差和大電流時會很熱。
- DC-DC 效率高，但外圍和噪聲議題通常比較多。

### 13.1 穩壓器是什麼

穩壓器的工作就是把輸入電源整理成系統真正需要的固定電壓。

例如：

- 12V 變 5V
- 5V 變 3.3V
- 電池電壓變成 MCU 可穩定使用的電壓

### 13.2 為什麼穩壓器很重要

因為大多數 IC 都不是什麼電都能吃。

你如果把：

- 12V 直接灌進 3.3V MCU
- 不穩的馬達電源直接拿來餵感測器

系統通常不會跟你客氣。

### 13.3 LDO 是什麼

LDO 是線性穩壓器的一種。  
它用比較直接的方法，把多出來的電壓變成熱。

所以它的好處是：

- 簡單
- 外圍常不多
- 輸出通常比較乾淨

但代價是：

- 壓差越大越熱
- 電流越大越熱
- 效率常不高

### 13.4 LDO 為什麼會熱

這是非常重要的基本計算。

```text
Ploss = (Vin - Vout) x Iout
```

例如：

- 12V 降到 5V
- 負載 300mA

功耗就是：

```text
(12 - 5) x 0.3 = 2.1W
```

2.1W 對很多小封裝 LDO 來說已經非常熱。  
這就是為什麼「能降壓」不代表「適合這樣降」。

### 13.5 什麼是 dropout

LDO 不是魔法。  
如果輸入電壓太接近輸出電壓，它也穩不住。

例如某顆 LDO 需要至少比輸出高 0.2V 到 0.5V 才能穩壓，那你要輸出 3.3V，就不能只給它 3.35V 還期望完全穩。

### 13.6 DC-DC 是什麼

DC-DC 是開關式電源轉換器。

它的核心不是把多餘電壓直接燒成熱，而是透過：

- 開關動作
- 電感儲能
- 電容平滑

把能量有效率地轉成你需要的電壓。

### 13.7 DC-DC 為什麼效率高

因為它盡量讓功率元件工作在：

- 很導通
- 或很關斷

減少停在又有壓降又有大電流的中間狀態。  
所以整體發熱通常比線性穩壓少很多。

### 13.8 LDO 與 DC-DC 怎麼選

很白話的選法：

如果：

- 壓差不大
- 電流不大
- 想簡單
- 想低噪聲

可以先考慮 LDO。

如果：

- 壓差大
- 電流大
- 在意效率
- 在意發熱

通常先考慮 DC-DC。

### 13.9 為什麼有時會 DC-DC 後面再接 LDO

這是很常見的工程架構。

例如：

```text
12V -> DC-DC -> 5V -> LDO -> 3.3V 類比 / MCU
```

原因是：

- 前面用 DC-DC 先把大壓差高效率降下來
- 後面用 LDO 再做比較乾淨的穩壓

這樣常能同時兼顧：

- 效率
- 發熱
- 輸出品質

### 13.10 為什麼穩壓器幾乎都需要電容

因為電容常是穩壓器穩定控制回路的一部分，不是可有可無的裝飾品。

沒照資料表放對：

- 容值
- ESR
- 擺放位置

可能會發生：

- 振盪
- 輸出不穩
- 負載一變就掉壓

### 13.11 什麼時候一定不要硬上 LDO

以下情況要提高警覺：

- 24V 降 5V，還要供幾百 mA
- 電池供電非常在意效率
- 負載電流很大
- 板子空間小、散熱差

這時很多時候 LDO 不是不能用，而是不划算、很熱、可靠度差。

### 13.12 穩壓器最常見錯誤

- 輸入輸出接反
- 只看輸出電壓，不看最大電流
- 忽略發熱
- 忽略 dropout
- 不照資料表放電容
- 把馬達和敏感電路混在同一個髒電源上

### 13.13 穩壓器這章一句話總結

穩壓器不是只要「數字對」就行，工程上真正要看的，是效率、熱、穩定度、雜訊和系統整體供電架構。

## 14. 開關與按鍵

### 14.0 先抓三件事

- 按鍵不是接上去就好，輸入腳不能浮動。
- 內建上拉是很適合新手的接法。
- 按鍵彈跳是正常現象，不是你特別倒楣。

### 14.1 按鍵是什麼

按鍵就是瞬間型開關。  
按下時導通，放開時回復。

### 14.2 為什麼按鍵一定要講上拉 / 下拉

因為 MCU 輸入腳不是神奇讀心器。  
如果一條輸入線既沒有被拉高、也沒有被拉低，它就可能受雜訊影響而亂跳。

### 14.3 常見接法：內建上拉

```text
GPIO 啟用內建上拉
GPIO -> 按鍵 -> GND
```

這時：

- 沒按：高
- 按下：低

這種接法很常見，因為簡單又省外部電阻。

### 14.4 機械按鍵彈跳是什麼

按鍵不是一下子就乾淨切換。  
它在按下和放開時，接點會抖一下，所以 MCU 可能看到：

- 高低高低高低

在幾毫秒內來回切很多次。

### 14.5 怎麼處理彈跳

- 軟體 debounce：最常見
- RC 濾波：硬體輔助
- Schmitt trigger：更穩定的邊界整形

### 14.6 按鍵最常見錯誤

- 沒做上拉 / 下拉
- 忘記 debounce
- 接錯腳，導致其實一直短路到 VCC 或 GND

### 14.7 按鍵這章一句話總結

按鍵很小，但它其實把數位輸入最基本的三件事全包了：預設狀態、邊界穩定、軟硬體配合。

## 15. 繼電器

### 15.0 先抓三件事

- 繼電器有兩個世界：線圈端和接點端。
- 線圈端通常要靠 BJT 或 MOSFET 驅動，還常要加 flyback diode。
- 要切 AC、要隔離、不要高頻 PWM 時，繼電器常很有價值。

### 15.1 繼電器是什麼

繼電器是用線圈產生磁力，去驅動機械接點切換的元件。

最白話地說：

> 你先用小電路去吸動它的機械開關，再用那個機械開關去切比較大的負載。

### 15.2 繼電器有兩個世界

這點非常重要。

第一個世界是線圈端：

- 它是電磁元件
- 會吃電流
- 需要驅動
- 關閉時會有反向尖峰

第二個世界是接點端：

- 它像機械開關
- 真正接觸你的負載
- 有接點壽命、額定電流、額定電壓

很多新手把這兩邊混成一團，所以很容易接錯。

### 15.3 線圈端怎麼驅動

通常不是 MCU 直接推，而是：

```text
MCU -> BJT 或 MOSFET -> 繼電器線圈
線圈兩端再加 flyback diode
```

因為線圈通常：

- 電流超過 GPIO 能力
- 是感性負載

### 15.4 接點端常見名詞

- COM：共同端
- NO：Normally Open，未吸合時斷開
- NC：Normally Closed，未吸合時接通

### 15.5 繼電器有什麼優點

- 可以切 AC
- 可以切高壓
- 控制端與負載端可隔離
- 關掉後漏電通常很低
- 對很多傳統負載很直覺

### 15.6 繼電器有什麼缺點

- 體積較大
- 有機械壽命
- 切換速度慢
- 不能高頻 PWM
- 線圈本身耗電
- 接點切感性負載時可能有火花和磨損

### 15.7 實際工程中怎麼在繼電器與電晶體之間選

這是很重要的決策題。

如果你要切的是：

- 220VAC
- 市電負載
- 大電壓差異的負載
- 需要明確隔離

通常先考慮繼電器或 SSR。

如果你要切的是：

- 12V 風扇
- 24V 電磁閥
- LED 燈條
- 馬達 PWM

通常優先考慮 MOSFET，不會先用繼電器。

如果你只是：

- 小電流控制
- 簡單 on/off
- 成本敏感

BJT 也很常夠用。

### 15.8 什麼時候繼電器比 MOSFET 更合理

- 你要切 AC
- 要乾淨隔離
- 不需要高速切換
- 負載形式很多變，想用機械接點通吃

### 15.9 什麼時候 MOSFET 比繼電器更合理

- 要 PWM
- 開關頻繁
- 想降低體積
- 想降低控制功耗
- 想減少機械壽命問題

### 15.10 繼電器最常見錯誤

- 直接 GPIO 推線圈
- 沒加 flyback diode
- 把接點額定值當成什麼都能切
- 把線圈端和接點端搞混
- 切感性大負載卻沒考慮接點壽命和火花

### 15.11 繼電器這章一句話總結

繼電器是很有價值的元件，但它不是萬能大開關；只要牽涉 AC、隔離、機械接點特性，你就要真的懂它兩個世界都在做什麼。

---

## 16. 蜂鳴器

### 16.0 先抓三件事

- 有源蜂鳴器給電就叫，無源蜂鳴器要吃頻率。
- 你要的是「發出提示音」還是「能控制音調」，這會直接決定選型。
- 不確定能不能直推時，先假設 GPIO 不該硬推會比較安全。

### 16.1 蜂鳴器分兩類

- 有源蜂鳴器：給 DC 電就會發出固定聲音
- 無源蜂鳴器：要餵頻率訊號才會叫

### 16.2 有源蜂鳴器的特性

優點：

- 控制超簡單
- 只要 on/off 就能發聲
- 很適合警報與提示音

缺點：

- 音調通常固定
- 可玩性低
- 不適合播旋律

### 16.3 無源蜂鳴器的特性

優點：

- 可以用 PWM 或方波控制不同頻率
- 可做旋律
- 可做不同提示音

缺點：

- 控制較麻煩
- MCU 需要提供頻率訊號

### 16.4 實際工程中怎麼選有源或無源

如果需求是：

- 開機叫一下
- 錯誤警報
- 操作提示
- 不在意音高變化

通常有源就很好用。

如果需求是：

- 多種提示音
- 播簡單旋律
- 想用不同頻率表達不同狀態

通常選無源。

### 16.5 蜂鳴器可以直接接 GPIO 嗎

要看型號和電流。

有些小型無源蜂鳴器或壓電片，GPIO 可能能直接驅動一點聲音。  
但工程上更穩健常是：

- 用 BJT
- 或用 MOSFET

這樣可避免：

- GPIO 過載
- 音量不足
- 啟動瞬間不穩

### 16.6 有源與無源最容易搞錯什麼

最常見錯誤是：

- 拿有源蜂鳴器去送 PWM，結果發聲怪怪的
- 拿無源蜂鳴器只給固定高電位，結果幾乎不叫

所以第一步一定要先確認型號。

### 16.7 蜂鳴器這章一句話總結

蜂鳴器不難，但有源和無源控制邏輯完全不同，工程上要先搞清楚你要的是「會叫」還是「能控制音調」。

## 17. 馬達

### 17.0 先抓三件事

- 馬達不是乖負載，它很常反過來把系統搞亂。
- 供電、驅動器、保護、電容，通常比「程式寫得多漂亮」更先決定能不能穩。
- 如果一開馬達 MCU 就重開機，先查電源，不要先怪程式。

### 17.1 馬達是什麼

馬達是把電能變成機械動作的元件。

但對電路設計者來說，更重要的是：

> 馬達不是乖負載，它是會反過來影響你整個系統的元件。

### 17.2 常見種類

- DC 馬達
- 步進馬達
- 伺服馬達
- BLDC

### 17.3 為什麼馬達會讓電路變難

因為它常同時帶來：

- 啟動大電流
- 持續變動負載
- 反電動勢
- 切換雜訊
- 供電壓降
- 地線彈跳

所以一顆馬達不只是吃電而已，它也常把髒東西丟回系統裡。

### 17.4 啟動大電流是什麼

馬達剛開始轉的瞬間，還沒有建立穩定反電動勢，所以電流常很大。  
這個電流可能比正常運轉電流大很多倍。

結果就是：

- 電源瞬間掉壓
- 線路發熱
- 驅動器超載
- MCU brown-out reset

### 17.5 反電動勢是什麼

馬達在轉動時，本身也像一個發電機。  
它會產生與供電相反方向的電壓，這就是反電動勢。

這件事會影響：

- 穩態電流
- 關閉瞬間波形
- 驅動器保護設計

### 17.6 為什麼馬達會讓 MCU 重開機

典型原因有這些：

- 啟動時把供電拉低
- 地線電位被大電流拉動
- 刷式馬達產生火花和高頻雜訊
- 保護和去耦做不夠

所以很多人會說：

> 明明程式沒問題，一開馬達 MCU 就死掉

這通常不是「馬達神秘」，而是供電和佈線被打爆了。

### 17.7 馬達旁邊要不要加電容

很多情況下，要。

但要先講清楚：  
電容不是萬能藥，它是抑制問題的一部分。

常見做法有：

- 驅動器供電端放大容量電容
- 馬達附近放去耦或抑制雜訊電容
- MCU 旁邊保持完整去耦

### 17.8 為什麼要在驅動器供電端放大電容

因為馬達吃電不是平穩小口小口吃，而常是大口抽。

這顆大電容的作用：

- 幫忙扛啟動瞬間需求
- 降低供電線路壓降
- 減少電源晃動傳回整板

常見會看到：

- `47uF`
- `100uF`
- `220uF`
- 更大，視馬達功率而定

### 17.9 馬達端子上為什麼有時也會加小電容

特別是刷式 DC 馬達，刷子接觸換向時會產生高頻火花雜訊。  
在馬達端子或外殼附近加小電容，有時能幫助降低高頻干擾。

常見做法是：

- 端子對端子加小陶瓷電容
- 端子對外殼各加一顆

但這要看馬達結構、EMI 需求和實驗結果，不是固定公式。

### 17.10 馬達要不要獨立電源

很常見答案是：

> 最好至少把馬達供電路徑和 MCU 供電路徑分開規劃。

不一定非得兩顆完全獨立電池，但你至少要思考：

- 馬達電流走哪條線
- MCU 和感測器吃的是哪一條穩壓路
- 兩者在哪裡共地

### 17.11 什麼叫做「分開供電」比較對

很多人以為分開供電就是完全不共地。  
這通常反而會出事。

比較好的說法是：

- 大電流負載和敏感電路不要共用細小、長、亂的供電路徑
- 電源分支要合理
- 地線回流路徑要規劃
- 最終仍常需要共地作為訊號參考

### 17.12 馬達供電的超具體建議

如果你用 MCU 控小直流馬達，先照這個基本原則做：

1. 馬達不要直接接 GPIO
2. 用 MOSFET、H 橋或專用驅動器
3. MCU 和驅動器要共地
4. 驅動器電源端放一顆大電容
5. MCU 3.3V 或 5V 旁邊保留自己的去耦
6. 馬達大電流回路不要和 MCU 地線共走細線太長
7. 線圈或感性回路做好 flyback 或驅動保護

### 17.13 什麼時候建議獨立電源更強烈

如果你遇到以下情況，就非常建議把馬達和控制系統電源切開管理：

- 馬達啟動電流很大
- 多顆馬達同時動
- MCU 很容易 brown-out
- 類比感測器精度要求高
- 無線通訊很怕雜訊

做法常是：

- 一路電源供馬達
- 一路經穩壓後供 MCU / 感測器

最終在合適位置共地。

### 17.14 馬達除了電容還要什麼保護

視拓樸不同，常還要：

- Flyback diode
- TVS
- MOSFET body diode 規劃
- H 橋死區控制
- 電流偵測
- 保險絲

### 17.15 馬達最常見錯誤

- 直接用 GPIO 推馬達
- 沒估啟動電流
- 沒有供電緩衝電容
- 沒有把大電流回路和敏感回路分開
- 忘記共地
- 用很弱的 USB 電源硬帶馬達又帶 MCU

### 17.16 馬達這章一句話總結

馬達最大的難點不是「讓它轉」，而是「讓它轉的同時整個系統還穩」。

## 18. 感測器

### 18.0 先抓三件事

- 感測器的工作，是把外界狀態翻成電訊號。
- 第一件要分清楚的是：它是類比輸出還是數位輸出。
- 感測器數值怪怪的，很多時候先查供電和介面，不是先查演算法。

### 18.1 感測器是什麼

感測器是把現實世界的物理量轉成電訊號的元件或模組。

它量的可能是：

- 光
- 溫度
- 壓力
- 距離
- 聲音
- 加速度
- 磁場
- 濕度
- 氣體濃度

### 18.2 感測器不是只有一顆頭而已

很多新手會把感測器當成一顆「會直接告訴你答案」的黑盒子。  
但實際上它常由很多部分組成：

- 感測元件本體
- 前端放大或調理電路
- 濾波
- ADC
- 數位介面
- 校正演算法

所以你買到的模組越高級，裡面通常幫你包了越多事情。

### 18.3 感測器可以怎麼分類

最實用的分類法有兩種。

第一種，按輸出型態分：

- 類比輸出
- 數位輸出
- 開關型輸出

第二種，按量測物理量分：

- 光學
- 溫度
- 壓力
- 距離
- 慣性
- 磁性
- 生物 / 化學

### 18.4 類比感測器是什麼

它輸出的是一個連續變化的電壓或電流。  
例如：

- 光敏電阻配分壓
- 熱敏電阻配分壓
- 類比壓力感測器
- 某些電流感測器

優點：

- 結構直觀
- 成本可低

缺點：

- 容易受雜訊影響
- 需要 ADC
- 常需要前端調理

### 18.5 數位感測器是什麼

它會把量測結果先在模組內處理，再透過數位介面輸出，例如：

- I2C
- SPI
- UART
- One-wire

優點：

- 抗雜訊通常較好
- MCU 直接讀資料比較方便
- 常內建校正與補償

缺點：

- 通訊更複雜
- 要懂資料格式和初始化流程
- 有位址、時序、電平匹配問題

### 18.6 開關型感測器是什麼

有些感測器輸出不是連續值，而是單純：

- 有 / 沒有
- 高 / 低
- 觸發 / 未觸發

例如：

- 某些接近開關
- 限位開關
- 光遮斷模組
- 比較器型輸出模組

### 18.7 感測器實際整合時要看什麼

這裡很重要，因為很多問題不是感測器壞，而是整合方式不對。

你要看：

- 供電電壓
- 邏輯電平
- 輸出型態
- 量測範圍
- 精度
- 解析度
- 取樣速度
- 反應時間
- 校正需求
- 環境耐受性

### 18.8 感測器為什麼常要搭配電阻與電容

因為很多感測器不會直接吐出完美訊號。

電阻常拿來：

- 分壓
- 偏壓
- 上拉
- 設定增益

電容常拿來：

- 濾波
- 穩定供電
- 抑制高頻雜訊

### 18.9 什麼感測器最適合初學者

如果你想練最基本觀念，推薦順序是：

1. 按鍵
2. 光敏電阻
3. 熱敏電阻
4. 超音波距離模組
5. I2C 溫濕度感測器
6. IMU

這樣你會一路學到：

- GPIO
- ADC
- 分壓
- 濾波
- 時序量測
- I2C 通訊

### 18.10 光敏電阻與熱敏電阻為什麼這麼適合教學

因為它們會直接讓你看到：

- 阻值會隨外界改變
- 要靠分壓把變化轉成電壓
- ADC 才能讀出來

這是新手建立「物理量 -> 電訊號 -> MCU 數值」觀念的超好入口。

### 18.11 類比感測器最常踩的坑

- ADC 參考電壓搞不清楚
- 訊號阻抗太高
- 沒加濾波
- 供電很髒
- 量測範圍超過 ADC 腳位

### 18.12 數位感測器最常踩的坑

- I2C 上拉沒處理好
- 邏輯電平不匹配
- 初始化順序錯
- 通訊速度設太快
- 模組供電和主控共地沒做好

### 18.13 工程上怎麼選感測器

不要只看「能不能量到」。

你還要問：

- 精度要多少
- 更新速度要多少
- 工作環境是室內還是工業現場
- 有沒有震動、灰塵、水氣、電磁干擾
- 後端 MCU 是否有足夠資源處理

### 18.14 感測器這章一句話總結

感測器真正難的不是買到一顆，而是把它供好電、接對介面、整理好訊號，最後讓 MCU 讀到可信的資料。

## 19. MCU 微控制器

### 19.0 先抓三件事

- MCU 是大腦，不是大電流驅動器。
- GPIO、ADC、PWM、UART、I2C、SPI 是你最常碰到的幾種能力。
- 只要牽涉馬達、繼電器、燈條、加熱片，通常都要外部驅動。

### 19.1 MCU 是什麼

MCU 是控制核心。它負責：

- 讀輸入
- 算邏輯
- 控輸出
- 跟外部裝置通訊

### 19.2 常見 MCU 平台

- STM32
- AVR / Arduino UNO
- ESP32
- RP2040

### 19.3 MCU 常見外設

- GPIO：一般輸入輸出
- ADC：量類比電壓
- PWM：輸出控制脈波
- UART：序列通訊
- I2C：感測器常用
- SPI：較高速周邊常用

### 19.4 MCU 不是萬能驅動器

這句很重要：

> MCU 是大腦，不是肌肉。

它很適合做決策，但碰到馬達、繼電器、燈條、加熱片這類負載，通常要靠外部驅動元件幫忙。

### 19.5 MCU 最常見限制

- 腳位電壓範圍
- 單腳最大電流
- 整體 I/O 總電流
- 5V tolerance 與否

### 19.6 MCU 為什麼很依賴穩定供電

供電不穩時，最常看到的不是「完全沒電」，而是：

- 重開機
- 程式亂跳
- ADC 不穩
- 通訊異常

### 19.7 MCU 這章一句話總結

MCU 很重要，但它只是系統的一部分；如果你把它當萬能供電腳和萬能驅動器，系統通常會出問題。

## 20. GND、電源、訊號到底怎麼分

### 20.0 先抓三件事

- GND 不只是負極，它還是很多訊號的共同參考點。
- 電源線是在送能量，訊號線是在送命令。
- 很多新手問題，其實是把這兩種線混著想造成的。

### 20.1 GND 為什麼不只是負極

因為它還常是系統共同參考點。  
很多訊號的高低，都是相對於 GND 來定義。

### 20.2 電源線是什麼

主要負責提供能量的線。  
例如：

- 12V 給風扇
- 5V 給模組
- 3.3V 給 MCU

### 20.3 訊號線是什麼

主要負責傳遞命令或狀態的線。  
例如：

- GPIO
- UART
- I2C
- SPI
- PWM 控制線

### 20.4 為什麼要把這兩種線分開想

因為很多初學者把「有連到 MCU」就當成「可以拿來供電」。  
這是錯的。GPIO 通常是訊號腳，不是大電流電源腳。

### 20.5 這章一句話總結

看電路時，先分清楚哪些線在送能量，哪些線在送命令，整體會清楚很多。

---

## 21. 類比與數位

### 21.0 先抓三件事

- 類比是連續變化，數位通常是高 / 低兩種狀態。
- 這個區分會直接影響你該接 ADC 還是 GPIO / 通訊介面。
- 類比訊號通常比數位訊號更怕雜訊。

### 21.1 數位訊號

比較在乎：

- 高 / 低
- 0 / 1
- 邏輯狀態

### 21.2 類比訊號

比較在乎連續變化的電壓或電流。  
例如光線、溫度、聲音、旋鈕位置。

### 21.3 為什麼這個區分重要

因為：

- 類比訊號常接 ADC
- 數位訊號常接 GPIO 或通訊介面
- 類比更容易受雜訊影響

### 21.4 類比與數位這章一句話總結

很多感測器問題的起點，不是程式，而是你一開始就沒分清楚自己手上的是類比還是數位訊號。

---

## 22. ADC

### 22.0 先抓三件事

- ADC 量的是電壓，不是萬物本身。
- 電壓一定要落在 MCU 允許的範圍內。
- 分壓和濾波，常常是 ADC 好不好用的前提。

### 22.1 ADC 是什麼

ADC = Analog to Digital Converter。  
白話講，就是把電壓變成 MCU 看得懂的數字。

### 22.2 ADC 最常搭配什麼

- 分壓電阻
- 濾波電容
- 感測器
- 運放前端

### 22.3 ADC 最常見錯誤

- 輸入電壓超範圍
- 沒做分壓就直接量高電壓
- 前端太吵，量測亂跳

### 22.4 ADC 這章一句話總結

ADC 不是萬能量測口，它只在正確電壓範圍、正確前端條件下才會量得好。

---

## 23. PWM

### 23.0 先抓三件事

- PWM 雖然是數位脈波，但常拿來做近似類比控制。
- LED 調光、風扇調速、無源蜂鳴器發聲，都很常用它。
- 如果你想拿繼電器接高頻 PWM，通常方向就錯了。

### 23.1 PWM 是什麼

PWM 是高低切換的脈波，但你可以控制它的占空比和頻率。

### 23.2 PWM 常用來做什麼

- LED 調亮度
- 風扇調速
- 無源蜂鳴器發聲
- 某些驅動器控制

### 23.3 PWM 這章一句話總結

PWM 看起來是數位脈波，但常被拿來做出近似類比的控制效果。

---

## 24. UART / I2C / SPI

### 24.0 先抓三件事

- 先分清楚你現在接的是 UART、I2C 還是 SPI，不要只看「都有幾條線」。
- UART 要注意 TX / RX 交叉，I2C 常要上拉，SPI 通常線比較多。
- 初學者很容易把通訊腳和電源腳混掉，這件事要一直提醒自己。

### 24.1 UART

常見兩條主要線：

- TX
- RX

很常拿來做 log、模組通訊、除錯。

### 24.2 I2C

常見兩條主要線：

- SDA
- SCL

感測器和小型周邊很常用。

### 24.3 SPI

常見線有：

- SCK
- MOSI
- MISO
- CS

速度通常較快，但線較多。

### 24.4 為什麼這章對元件入門也重要

因為很多模組不是「一條訊號線就結束」，而是透過通訊介面和 MCU 交換資料。

### 24.5 這章一句話總結

遇到模組時，先分清楚供電腳和通訊腳，不要看到「四五條線」就亂接。

---

## 25. 實際接線範例

### 25.1 LED + 電阻 + MCU

用途：狀態燈

```text
GPIO -> 330Ω -> LED 正極
LED 負極 -> GND
```

學到的重點：

- GPIO 輸出
- LED 極性
- 限流

### 25.2 按鍵 + 上拉 + MCU

用途：最基本輸入

```text
GPIO 啟用內建上拉
GPIO -> 按鍵 -> GND
```

學到的重點：

- 上拉
- active low 輸入
- debounce

### 25.3 光敏電阻 + 固定電阻 + ADC

用途：量亮度

```text
3.3V -> 光敏電阻 -> 中點 -> 固定電阻 -> GND
中點 -> ADC
```

學到的重點：

- 分壓
- 類比量測
- 感測器不是直接吐數字

### 25.4 電位器 + ADC

用途：量旋鈕位置

```text
3.3V -> 電位器一端
GND -> 電位器另一端
滑動端 -> ADC
```

學到的重點：

- 連續類比量測
- 分壓的直觀理解

### 25.5 MOSFET 控 12V 風扇

用途：用 MCU 控制較大負載

```text
12V -> 風扇 -> Drain
Source -> GND
GPIO -> 100Ω -> Gate
Gate -> 10kΩ -> GND
MCU GND 與 12V GND 共地
```

學到的重點：

- MOSFET 開關
- 共地
- 大負載不直接接 GPIO

### 25.6 BJT 控繼電器

```text
VCC -> 線圈 -> Collector
Emitter -> GND
GPIO -> 1kΩ -> Base
線圈兩端並聯 flyback diode
```

學到的重點：

- BJT 當開關
- 線圈保護
- MCU 與大負載間需要中介

### 25.7 LDO 供 3.3V MCU

```text
輸入電源 -> LDO IN
LDO OUT -> 3.3V 系統
IN 與 OUT 依資料表放電容
GND 全部相連
```

學到的重點：

- 穩壓器不是三隻腳插上就結束
- 電容常是必要條件

## 26. 新手除錯清單

### 26.1 系統完全沒反應

先查：

1. 有沒有供電
2. GND 有沒有接
3. 穩壓器輸出對不對
4. 板子有沒有上電跡象

### 26.2 LED 不亮

先查：

1. LED 方向
2. 有沒有串電阻
3. GPIO 腳位定義對不對
4. active high / low 邏輯有沒有看錯

### 26.3 按鍵亂跳

先查：

1. 有沒有上拉 / 下拉
2. 有沒有 debounce
3. 接線穩不穩

### 26.4 MOSFET 控不動負載

先查：

1. 有沒有共地
2. Gate 是否真的切換
3. MOSFET 是否適合 3.3V 驅動
4. Drain / Source 有沒有接反

### 26.5 ADC 值亂跳

先查：

1. 電源穩不穩
2. ADC 輸入是否超範圍
3. 有沒有分壓 / 濾波
4. 地參考是否合理

### 26.6 繼電器不吸合

先查：

1. 線圈電壓對不對
2. 驅動電晶體是否工作
3. 線圈旁邊有沒有保護二極體

### 26.7 除錯的核心心法

不要一開始就改程式十幾個地方。  
先量供電，再看接線，再看邏輯，再看程式。

## 27. 元件選型入門

### 27.1 選電阻

看：

- 阻值
- 功率
- 精度
- 封裝

### 27.2 選電容

看：

- 容值
- 額定電壓
- 是否有極性
- 材質

### 27.3 選 MOSFET

看：

- 最大電壓
- 最大電流
- `Rds(on)`
- 是否適合 3.3V Gate 驅動
- 封裝散熱

### 27.4 選穩壓器

看：

- 輸入範圍
- 輸出電壓
- 最大電流
- 散熱
- 是否需要哪些外部元件

### 27.5 選繼電器

看：

- 線圈電壓
- 接點額定
- AC / DC 負載條件

### 27.6 選感測器

看：

- 工作電壓
- 輸出型式
- 量測範圍
- 是否容易和你的 MCU 整合

### 27.7 選型一句話心法

不是「能用就好」，而是「在你的電壓、電流、邏輯條件下能穩定工作」。

### 第一階段：最基本元件

- LED

練習：

### 第二階段：ADC 與感測器

- 電位器

## 29. 資料表怎麼看

初學者不用一次把資料表全啃完。先看這幾塊最有用：

1. 腳位圖
2. 絕對最大額定
3. 建議工作條件
4. 典型應用電路
5. 重要電氣特性

最常見錯誤是把「最大值」當成「平常建議值」。  
不是能撐到 40V，就代表你平常放 39V 最舒服。

典型應用電路也很值得看，因為它常直接告訴你：

- 哪些電阻電容是必要的
- 哪個接法才是標準用法

## 30. 逐章補充：實際接線圖與小練習題

這一章不是新觀念，而是把前面主文裡的重點變成「你真的可以拿麵包板或模組做出來」的版本。  
我刻意用最小可行接法來畫，讓你能快速練到核心觀念。

### 30.1 第 4 章：電池與電源

實際接線圖：

```text
12V 變壓器 + ------> Buck IN+
12V 變壓器 - ------> Buck IN-

Buck OUT+ ---------> 5V 模組 / 開發板 VIN
Buck OUT- ---------> GND

5V 模組 / 開發板內部再產生 3.3V 給 MCU
所有 GND 共地
```

小練習題：

1. 如果你把 12V 直接接到 `3V3` 腳，最可能先壞的是什麼？
2. 為什麼馬達電源 GND、MCU GND、感測器 GND 常常都要接在一起？
3. 如果馬達一啟動 MCU 就重開機，你會先查電源的哪三件事？

### 30.2 第 5 章：電阻

實際接線圖：

```text
3.3V GPIO ----> 330Ω ----> LED 正極
LED 負極 ----> GND
```

小練習題：

1. 如果紅色 LED 順向電壓約 2.0V，你想讓它流 5mA，3.3V 系統大約該選多少電阻？
2. 為什麼 `10kΩ` 常拿來做上拉，而不是 `100Ω`？
3. 如果你把 `330Ω` 改成 `1kΩ`，亮度和電流大概會怎麼變？

### 30.3 第 6 章：電容

實際接線圖：

```text
5V 電源 -------> 板子 5V 入口
GND -----------> 板子 GND

板子 5V 入口附近：10uF 電解電容 到 GND
MCU VDD 腳旁邊：0.1uF 陶瓷電容 到 GND
```

小練習題：

1. 為什麼 `0.1uF` 去耦電容要貼近 MCU 電源腳，而不是放在板子另一端？
2. 如果電解電容接反，實際上可能看到什麼現象？
3. 為什麼很多板子會同時放一顆大電容和一顆小電容？

### 30.4 第 7 章：電感

實際接線圖：

```text
12V 電源 + -----> Buck 模組 IN+
12V 電源 - -----> Buck 模組 IN-

Buck 模組 OUT+ -> 5V 負載
Buck 模組 OUT- -> GND
```

說明：這裡雖然你看不到模組內部的電感，但真正在做高效率降壓的核心就是它。

小練習題：

1. 為什麼 12V 降 5V、電流又不小時，很多人會先選 Buck 而不是只靠電阻？
2. 電感在開關電源裡比較像是在處理電壓，還是處理電流變化？
3. 如果你完全不懂 DC-DC 細節，至少該先知道模組的哪三個規格？

### 30.5 第 8 章：二極體

實際接線圖：

```text
VCC ----> 線圈 ----> 晶體管開關 ----> GND
          |                     |
          +----|<|-------------+
               Flyback diode
```

小練習題：

1. 為什麼 flyback diode 平常通電時通常不導通？
2. 為什麼線圈關閉瞬間會冒出高壓尖峰？
3. 如果這顆二極體方向接反，可能出現什麼結果？

### 30.6 第 9 章：LED

實際接線圖：

```text
3.3V ----> 330Ω ----> LED 正極
LED 負極 ----> GPIO

GPIO 輸出 Low 時，LED 亮
GPIO 輸出 High 時，LED 滅
```

說明：這是 active-low 接法，很多板載 LED 就是這樣。

小練習題：

1. 為什麼同樣是 LED，有些板子寫 `GPIO = 0` 反而會亮？
2. 白色 LED 和紅色 LED 的限流電阻為什麼常不一樣？
3. 如果 LED 完全不亮，你會先查極性、電阻，還是 GPIO 邏輯？

### 30.7 第 10 章：BJT

實際接線圖：

```text
5V ----> 蜂鳴器 ----> Collector
Emitter ----> GND
GPIO ----> 1kΩ ----> Base
```

小練習題：

1. 為什麼 Base 前面要串一顆電阻？
2. 如果蜂鳴器電流很小，BJT 相比 MOSFET 的優勢可能是什麼？
3. 如果把 Collector 和 Emitter 接反，常會出現什麼怪現象？

### 30.8 第 11 章：MOSFET

實際接線圖：

```text
12V ----> 風扇 ----> Drain
Source ----> GND
GPIO ----> 100Ω ----> Gate
Gate ----> 10kΩ ----> GND
MCU GND 與 12V GND 共地
```

小練習題：

1. 為什麼 Gate 要加下拉電阻？
2. 為什麼只看 `Vgs(th)` 不夠？
3. 如果 MCU 沒和 12V 電源共地，最可能看到什麼問題？

### 30.9 第 12 章：放大器 / 運算放大器

實際接線圖：

```text
電位器滑動端 ----> 運放 +
運放輸出 --------> 運放 -
運放輸出 --------> MCU ADC
運放供電：3.3V 與 GND
```

說明：這是最典型的 buffer，目的是讓 ADC 看到更穩定的訊號源。

小練習題：

1. 如果不做 buffer，某些高阻抗感測器直接接 ADC 可能會有什麼問題？
2. 為什麼很多 3.3V 系統會在意運放能不能 rail-to-rail？
3. 放大器一定是把訊號變大嗎？

### 30.10 第 13 章：穩壓器

實際接線圖：

```text
12V ----> Buck ----> 5V ----> LDO ----> 3.3V MCU
           |                    |
          GND ------------------+

Buck 與 LDO 的輸入輸出端，依資料表放電容
```

小練習題：

1. 為什麼 12V 轉 3.3V、電流又大時，常先 Buck 再 LDO？
2. 如果一顆 LDO 從 12V 降到 5V、輸出 500mA，為什麼要特別注意發熱？
3. 資料表寫輸出端要放電容，這顆電容是可有可無的嗎？

### 30.11 第 14 章：開關與按鍵

實際接線圖：

```text
GPIO 啟用內建上拉
GPIO ----> 按鍵 ----> GND
```

小練習題：

1. 沒按按鍵時，GPIO 應該讀到高還是低？
2. 為什麼按一下按鍵，程式可能判成按了很多次？
3. 軟體 debounce 和 RC debounce 有什麼差別？

### 30.12 第 15 章：繼電器

實際接線圖：

```text
MCU GPIO -> 1kΩ -> NPN Base
NPN Emitter -> GND
NPN Collector -> 繼電器線圈一端
線圈另一端 -> 5V
線圈兩端並聯 flyback diode

12V 負載 + -> COM
NO -> 燈 / 負載 -> 12V 負載 -
```

小練習題：

1. 線圈端和接點端哪一邊是真正去切負載？
2. 如果你要切 AC 風扇，為什麼通常先想到繼電器而不是單顆 N-MOS？
3. 如果你要高速 PWM 控制負載，為什麼繼電器通常不適合？

### 30.13 第 16 章：蜂鳴器

實際接線圖：

```text
5V ----> 有源蜂鳴器 ----> Collector
Emitter ----> GND
GPIO ----> 1kΩ ----> Base
```

如果是無源蜂鳴器，常把 GPIO 或 PWM 腳接到驅動級，再用不同頻率發聲。

小練習題：

1. 什麼情況下有源蜂鳴器會比無源蜂鳴器更省事？
2. 如果你想播簡單旋律，應該優先選哪一種？
3. 為什麼有些蜂鳴器直接給高電位幾乎沒聲音？

### 30.14 第 17 章：馬達

實際接線圖：

```text
12V ----+----> 馬達 ----> Drain
        |
        +----> 220uF 電解 ----> GND

Source ----> GND
GPIO ----> 100Ω ----> Gate
Gate ----> 10kΩ ----> GND
馬達兩端或驅動路徑依拓樸加保護
MCU GND 與 12V GND 共地
```

小練習題：

1. 為什麼馬達啟動時比穩定轉動時更容易讓電源掉壓？
2. 大電容應該更靠近電源入口，還是更靠近馬達驅動器？
3. 什麼情況下你會很想把馬達供電和 MCU 供電分支開來？

### 30.15 第 18 章：感測器

實際接線圖一：類比感測器

```text
3.3V ----> 光敏電阻 ----> 中點 ----> 固定電阻 ----> GND
中點 ----> MCU ADC
```

實際接線圖二：數位感測器

```text
3.3V ----> I2C 感測器 VCC
GND ----> I2C 感測器 GND
SCL ----> MCU SCL
SDA ----> MCU SDA
SCL / SDA 視需要加上拉
```

小練習題：

1. 光敏電阻這種做法，為什麼一定要搭固定電阻才能量？
2. 數位感測器如果是 3.3V 邏輯，能不能直接接 5V MCU，要先查什麼？
3. 感測器數值亂跳時，你會先懷疑供電、訊號還是程式？

### 30.16 第 19 章：MCU

實際接線圖：

```text
3.3V ----> MCU VDD
GND ----> MCU VSS
0.1uF ----> VDD 與 GND 之間，靠近晶片
NRST ----> 10kΩ ----> 3.3V
GPIO1 ----> LED + 電阻 ----> GND
GPIO2 ----> 按鍵 ----> GND
UART TX/RX ----> USB-TTL
```

小練習題：

1. 為什麼 MCU 一定要有穩定的 VDD 與 GND，而不是只接一邊？
2. 如果程式燒進去了但完全沒反應，你會先查哪三條最基本的線？
3. 為什麼很多除錯都會先把 UART log 接出來？

### 30.17 第 20 章：GND、電源、訊號

實際接線圖：

```text
5V 電源 ------> 感測器 VCC
GND ----------> 感測器 GND
Signal -------> MCU GPIO / ADC

另一個大電流負載走自己的供電線
最終與 MCU 在合理位置共地
```

小練習題：

1. 哪些線是在送能量，哪些線是在送命令？
2. 為什麼不能把 GPIO 當成一個小電源腳來想？
3. 如果兩個模組訊號接了但沒共地，常會出現什麼症狀？

### 30.18 第 21 章：類比與數位

實際接線圖：

```text
電位器滑動端 ----> ADC
按鍵 ---------> GPIO
```

說明：同樣都是「輸入」，但一個是連續值，一個是高低狀態。

小練習題：

1. 為什麼電位器通常接 ADC，而不是一般數位輸入？
2. 為什麼按鍵通常不需要 ADC？
3. 什麼情況下類比訊號比數位訊號更容易受干擾？

### 30.19 第 22 章：ADC

實際接線圖：

```text
電池正極 ----> R1 ----> 中點 ----> R2 ----> GND
中點 ----> ADC
中點 ----> 0.1uF ----> GND
```

說明：這是很典型的分壓量電池電壓接法。

小練習題：

1. 為什麼不能把 12V 直接接到 3.3V MCU 的 ADC？
2. 中點加一顆小電容，常是為了改善什麼？
3. 如果 ADC 值上下亂跳，你會先檢查分壓、地線，還是取樣時間？

### 30.20 第 23 章：PWM

實際接線圖：

```text
PWM 腳 ----> 100Ω ----> MOSFET Gate
Gate ----> 10kΩ ----> GND
12V ----> LED 燈條 / 風扇 ----> Drain
Source ----> GND
```

小練習題：

1. 為什麼 PWM 能拿來做調光和調速？
2. 為什麼繼電器通常不拿來接高頻 PWM？
3. 如果負載不跟著變化，你會先懷疑 PWM 腳、Gate，還是共地？

### 30.21 第 24 章：UART / I2C / SPI

實際接線圖一：UART

```text
MCU TX ----> USB-TTL RX
MCU RX ----> USB-TTL TX
GND   ----> GND
```

實際接線圖二：I2C

```text
MCU SCL ----> 感測器 SCL
MCU SDA ----> 感測器 SDA
3.3V -------> 感測器 VCC
GND --------> 感測器 GND
SCL / SDA 經 4.7kΩ 上拉到 3.3V
```

實際接線圖三：SPI

```text
MCU SCK  ----> 模組 SCK
MCU MOSI ----> 模組 MOSI
MCU MISO ----> 模組 MISO
MCU CS   ----> 模組 CS
GND      ----> GND
```

小練習題：

1. 為什麼 UART 的 TX 和 RX 要交叉接，而不是同名對同名？
2. 為什麼 I2C 常需要上拉電阻？
3. 為什麼 SPI 通常比 I2C 線更多？

## 31. 資料表怎麼看實例版

這一章把第 29 章的原則，直接變成你真的會遇到的判讀流程。  
重點不是把資料表每一頁都唸一遍，而是學會：

- 該先找哪幾欄
- 哪些數字真的影響設計
- 哪些數字最容易被新手看錯

### 31.1 實例一：LED 資料表

假設你看到一顆紅色 LED 的資料表重點如下：

```text
Forward Current, If (recommended): 5mA ~ 20mA
Forward Voltage, Vf (typ): 2.0V @ If = 10mA
Forward Voltage, Vf (max): 2.4V @ If = 20mA
Reverse Voltage: 5V
```

你真正要抓的是：

- 建議工作電流大概在哪
- 順向電壓大概落在哪
- 反向不要亂接

如果你現在是 3.3V 系統，想讓它大約流 5mA，可以粗估：

```text
R = (3.3 - 2.0) / 0.005 = 260Ω
```

所以工程上就會去選：

- `270Ω`
- `330Ω`

而不是硬找 `260Ω`。

這裡最常見的誤讀是：

- 把最大電流當作平常就該這樣用
- 完全不看順向電壓範圍
- 以為反向接只是不亮，結果其實超規

### 31.2 實例二：MOSFET 資料表

假設你看到一顆 N-MOSFET 的重點如下：

```text
Drain-Source Voltage, VDS: 30V
Continuous Drain Current, ID: 8A
Gate Threshold Voltage, VGS(th): 1.0V ~ 2.0V
RDS(on): 25mΩ @ VGS = 4.5V
RDS(on): 40mΩ @ VGS = 2.5V
```

新手最容易只盯著：

```text
VGS(th) = 1.0V ~ 2.0V
```

然後就以為：

> 我的 3.3V GPIO 一定推得動。

這樣想太危險。  
你真正該在意的是：

- 在接近你實際 Gate 電壓時，它的 `RDS(on)` 多低
- 這個低阻值下，發熱能不能接受

例如你要帶 3A 負載，若 `RDS(on)` 取 `0.04Ω`：

```text
P = I^2 x R = 3^2 x 0.04 = 0.36W
```

0.36W 在小封裝上已經值得注意散熱。

這裡最常見的誤讀是：

- 把門檻電壓當成完全導通電壓
- 只看最大電流，不看溫度條件
- 不看封裝散熱能力

### 31.3 實例三：LDO 資料表

假設你看到一顆 3.3V LDO 的重點如下：

```text
Input Voltage Range: 2.5V ~ 6V
Output Voltage: 3.3V
Output Current: 300mA
Dropout Voltage: 250mV @ 300mA
Recommended CIN: 1uF
Recommended COUT: 1uF
```

你第一眼要問的不是「它能不能輸出 3.3V」，而是：

- 你的輸入有沒有落在範圍內
- 你的負載電流有沒有超過它
- 你的輸入和輸出壓差夠不夠
- 你有沒有照資料表放電容

例如：

- 用 5V 進、輸出 3.3V、負載 100mA：通常合理
- 用 12V 進：直接超出輸入範圍，不行

再例如如果它輸出 300mA，dropout 需要 250mV，  
那輸入至少要大約高於 3.3V 一截，不然輸出可能穩不住。

這裡最常見的誤讀是：

- 覺得「3.3V LDO」就能把任何高電壓變成 3.3V
- 忽略輸入上限
- 忽略輸出電容是穩定條件的一部分

### 31.4 實例四：繼電器資料表

假設你看到一顆繼電器的重點如下：

```text
Coil Voltage: 5V
Coil Resistance: 70Ω
Must Operate Voltage: 75% of nominal
Contact Rating: 10A 250VAC / 5A 30VDC
Operate Time: 10ms
Release Time: 5ms
```

這裡你至少要會看四件事：

1. 線圈要吃多少電流
2. 觸點能切什麼負載
3. AC 與 DC 額定不能混看
4. 切換速度是否符合需求

由線圈電阻可估：

```text
I = V / R = 5 / 70 ≈ 71mA
```

這就直接告訴你：

- MCU GPIO 幾乎不該直接推
- 你應該準備 BJT 或 MOSFET 驅動

`10A 250VAC` 不代表一定也能輕鬆切 `10A 30VDC`。  
因為 AC 和 DC 的滅弧條件不同，資料表往往分開列。

這裡最常見的誤讀是：

- 只看接點大電流，忘了線圈也要電流
- 把 AC 額定當成 DC 額定
- 完全沒注意 operate / release time

### 31.5 實例五：I2C 感測器資料表

假設你看到一顆 I2C 感測器的重點如下：

```text
Supply Voltage: 1.8V ~ 3.6V
Interface: I2C, up to 400kHz
I2C Address: 0x76 / 0x77
Startup Time: 2ms
Input Voltage on digital pin: VDD + 0.3V max
```

你第一眼就要先判斷：

- 這顆能不能直接吃你的 5V 或 3.3V
- 你 MCU 的 I2C 上拉應該拉到哪個電壓
- 位址會不會和別的裝置衝突
- 開機後是不是要等一下再讀

如果這顆只能吃到 `3.6V`，那你就不能把：

- VCC 直接接 5V
- SDA / SCL 也上拉到 5V

這裡最常見的誤讀是：

- 模組寫 I2C 就以為任何電壓都能接
- 忽略位址衝突
- 忘記 startup time，剛上電就急著讀

### 31.6 一份資料表，實際上最該先圈起來的欄位

不管是什麼元件，入門先抓這些：

1. 腳位圖
2. 絕對最大額定
3. 建議工作條件
4. 電氣特性
5. 典型應用電路
6. 必要外部元件

如果是功率元件，再加看：

1. 功耗
2. 熱阻
3. 封裝
4. 降額曲線

如果是通訊元件，再加看：

1. 介面電壓
2. 位址
3. 時序
4. 初始化需求

### 31.7 資料表判讀的實際流程

建議你每次都照這個順序：

1. 先看這顆元件在電路裡要扮演什麼角色
2. 再看它的供電條件能不能符合你的系統
3. 再看它的控制或輸出條件能不能和 MCU 相容
4. 再看需要哪些外部電阻電容
5. 最後才去看進階性能指標

### 31.8 資料表最常見的三種誤讀

1. 把「絕對最大額定」當「平常建議工作值」
2. 只看一個漂亮數字，不看測試條件
3. 只看功能，不看外部條件和限制

### 31.9 資料表實例版小練習題

1. 一顆 MOSFET 資料表只給 `RDS(on) @ 10V`，你的 MCU 是 3.3V，這時你最該警覺什麼？
2. 一顆 LDO 輸入上限 6V，你的系統是 2S 鋰電池，為什麼不能直接拿來用？
3. 一顆感測器供電範圍是 `1.8V ~ 3.6V`，如果你的 I2C 上拉到 5V，哪裡可能先出事？

## 32. 小練習題參考答案

這一章是第 30 章與第 31 章小練習題的參考答案。  
「參考答案」的意思不是只能這樣寫，而是你至少要抓到這個方向。

### 32.1 第 4 章：電池與電源

1. 最可能先壞的是直接掛在 `3V3` 電源軌上的元件，尤其是 MCU、本來只允許 3.3V 的 IC，甚至板上的穩壓與保護元件也可能一起受損。
2. 因為訊號高低一定要有共同參考點。沒有共地，UART、ADC、MOSFET 控制這些訊號都可能「看不懂彼此在講什麼」。
3. 先查電源本身電流能力夠不夠，再查啟動瞬間有沒有掉壓，最後查大電流回路、共地、以及驅動器附近是否有足夠的大電容。

### 32.2 第 5 章：電阻

1. 理論值約 `260Ω`，工程上常改選最接近的標準值，例如 `270Ω`，若想更保守也常直接用 `330Ω`。
2. 因為 `100Ω` 當上拉太強，會白白浪費很多電流；`10kΩ` 通常已經足夠把腳位穩定拉住，又不會太耗電。
3. 電流會下降，亮度通常也會跟著下降，但 GPIO 負擔會更小、整體更省電。

### 32.3 第 6 章：電容

1. 因為去耦要處理的是非常短、非常快的瞬間電流需求，離太遠會被走線寄生電感和阻抗拖慢效果。
2. 可能出現發熱、漏液、鼓起來、容量失效，嚴重時甚至直接損壞。
3. 因為大電容比較擅長扛比較大的能量變化，小電容比較擅長處理高頻快速雜訊，兩者任務不同。

### 32.4 第 7 章：電感

1. 因為只靠電阻降壓效率很差，而且負載一變電壓就不穩；Buck 則能用比較高效率把 12V 穩定降到 5V。
2. 比較像是在處理電流變化。電感最核心的個性就是不喜歡電流突然改變。
3. 至少先看輸入電壓範圍、輸出電壓、可提供的最大電流；若更仔細，還要看效率、發熱和紋波。

### 32.5 第 8 章：二極體

1. 因為正常通電時，這顆 diode 在那個方向上是逆向偏壓，所以它平常幾乎不工作。
2. 因為線圈本質上是電感，電感不喜歡電流被突然切斷，所以會自己產生電壓，試圖把原本的電流繼續推出去。
3. 如果方向接反，平常一上電就可能形成不該有的導通路徑，造成線圈無法正常動作、二極體發熱，甚至讓電源吃重。

### 32.6 第 9 章：LED

1. 因為那是 active-low 接法，GPIO 輸出 Low 時是在幫 LED 提供回到 GND 的電流路徑。
2. 因為不同顏色 LED 的順向電壓不同，在同樣供電下，落到電阻上的電壓就不同，所以限流值也常不同。
3. 實作上會先查極性和限流電阻，再查 GPIO 腳位與邏輯設定，因為前兩個是最常見、最基本的錯誤。

### 32.7 第 10 章：BJT

1. 因為 Base 不能直接硬灌，要用電阻限制基極電流，保護 MCU，也讓 BJT 進入可預期的工作狀態。
2. 在小電流場合，BJT 常更便宜、容易買、電路也很直觀，所以很多簡單 on/off 控制還是很好用。
3. 常見現象是好像「有點動、又不太對」，例如電流帶不動、壓降很大、容易發熱，整體表現會很怪。

### 32.8 第 11 章：MOSFET

1. 因為 MCU 還沒初始化或腳位浮動時，下拉電阻能把 Gate 固定在關閉狀態，避免負載自己亂開。
2. 因為 `Vgs(th)` 只代表它開始稍微導通，不代表它在 3.3V 下已經低損耗、足夠帶你要的電流。
3. 最常見的是開關不乾淨，像是負載完全不動、只半開、或行為很不穩，因為 Gate 的參考點不對。

### 32.9 第 12 章：放大器 / 運算放大器

1. 因為 ADC 取樣時會拉一點瞬間電流，高阻抗訊號源可能因此被拉歪，讀值就容易不穩。
2. 因為 3.3V 系統的輸入和輸出空間本來就不大，如果不能靠近電源軌，可用範圍就會更小。
3. 不一定。放大器也常拿來做緩衝、濾波、偏移和阻抗轉換。

### 32.10 第 13 章：穩壓器

1. 因為前段 Buck 可以先高效率降壓，後段 LDO 再把輸出整理得更乾淨，這樣能同時兼顧效率和穩定度。
2. 因為功耗會很大，約是 `(12 - 5) x 0.5 = 3.5W`，這對很多 LDO 來說已經很容易過熱。
3. 通常不是可有可無。很多穩壓器把輸出電容當成穩定工作的必要條件，不照放可能直接振盪或不穩。

### 32.11 第 14 章：開關與按鍵

1. 在內建上拉的接法下，沒按時應該讀到高。
2. 因為機械按鍵會彈跳，接點不會只切一次，而是會在短時間內抖好幾下。
3. 軟體 debounce 是程式判斷延遲與濾掉抖動；RC debounce 是用硬體低通先把抖動壓掉。

### 32.12 第 15 章：繼電器

1. 真正切負載的是接點端，不是線圈端。線圈端只是拿來吸合繼電器。
2. 因為切 AC 常需要機械接點或專用 AC 開關方式，單顆 N-MOS 不適合直接拿來像機械開關那樣切一般 AC 負載。
3. 因為繼電器是機械件，速度慢、壽命有限，而且高頻切換會很快把它操壞。

### 32.13 第 16 章：蜂鳴器

1. 當你只需要簡單提示音、警報音、開機叫一聲時，有源蜂鳴器最省事，因為給電就會叫。
2. 要播旋律或不同音高時，通常優先選無源蜂鳴器。
3. 很多這種情況是因為它其實是無源蜂鳴器，需要 PWM 或交流式切換訊號，不是只給一個固定高電位就會正常叫。

### 32.14 第 17 章：馬達

1. 因為啟動瞬間還沒有建立穩定反電動勢，馬達看起來比較像低阻抗負載，所以會猛拉大電流。
2. 一般會希望更靠近馬達驅動器，因為它的主要任務就是幫驅動器就近扛瞬間電流變化。
3. 當啟動電流很大、MCU 常 brown-out、類比量測很敏感、或多顆馬達同時動時，就很值得把兩邊供電分支開。

### 32.15 第 18 章：感測器

1. 因為光敏電阻本身只是在變阻值，還要搭一顆固定電阻，才能把阻值變化轉成 ADC 看得懂的電壓變化。
2. 要先查 MCU 腳位是否 5V tolerant、感測器邏輯電平範圍、以及是否需要 level shifter。
3. 除錯順序通常先查供電和訊號，再查程式，因為很多感測器「亂跳」其實先是硬體問題。

### 32.16 第 19 章：MCU

1. 因為 MCU 不只需要電壓，還需要穩定的地參考；沒有完整的 VDD 和 GND 回路，系統根本不可能正常工作。
2. 最基本先查 `VDD`、`GND`、`NRST` 這三條；若還不對，再進一步看 boot 設定和時鐘。
3. 因為 UART log 是最直接的「我程式到底有沒有跑到這裡」證據，能大幅縮小除錯範圍。

### 32.17 第 20 章：GND、電源、訊號

1. 像 `5V`、`12V`、`3V3` 和回流的 `GND` 是送能量；像 GPIO、UART、I2C、SPI 這些主要是在送命令或狀態。
2. 因為 GPIO 的設計目的是邏輯控制，不是供大電流用；把它當小電源腳，很容易超規。
3. 常見症狀是通訊不穩、數值亂跳、輸入誤判、MOSFET 控制異常，因為沒有共同參考點。

### 32.18 第 21 章：類比與數位

1. 因為電位器輸出的是連續變化電壓，不是單純高或低，所以比較適合接 ADC。
2. 因為按鍵本質上只是開或關，通常只需要一般 GPIO 判斷高低就夠了。
3. 當訊號源阻抗高、線路長、供電髒、或旁邊有大電流切換時，類比訊號通常比數位訊號更容易受干擾。

### 32.19 第 22 章：ADC

1. 因為 ADC 腳位通常只能吃到 MCU 供電範圍附近，12V 直接進去很可能立刻超規甚至傷到晶片。
2. 這顆小電容常拿來做簡單低通濾波，讓分壓點更穩，也讓 ADC 取樣瞬間比較不容易被拉歪。
3. 實作上會先查分壓比例和地參考是否正確，再看濾波和取樣時間，因為前面兩項更常是根本問題。

### 32.20 第 23 章：PWM

1. 因為 PWM 是快速開關，改變占空比就等於改變平均送到負載的能量，所以能調亮度或調速。
2. 因為繼電器是機械件，切換太快不但跟不上，還會加速磨損。
3. 我會先查 PWM 腳真的有沒有輸出波形，再查 Gate 是否有被正確驅動，最後確認共地是否完整。

### 32.21 第 24 章：UART / I2C / SPI

1. 因為一邊的 TX 是送資料出去，所以必須接到另一邊的 RX；如果 TX 接 TX，兩邊都只是在講話，沒人在聽。
2. 因為 I2C 很常是 open-drain 或 open-collector 結構，本身只會主動拉低，需要上拉電阻把線拉回高電位。
3. 因為 SPI 通常至少有時鐘、主送從收、從送主收、片選幾條線，它是用比較多線換取更直接和高速的資料交換。

### 32.22 第 31 章：資料表怎麼看實例版

1. 你最該警覺的是：這顆 MOSFET 可能沒有保證在 3.3V Gate 下能低阻、低熱地工作，所以不能因為它「看起來會動」就當成安全可用。
2. 因為 2S 鋰電池滿電時大約可到 `8.4V`，已經超過 6V 上限，直接接上去就有超規風險。
3. 最可能先出事的是感測器的數位 I/O 腳或其內部保護結構，因為 5V 已經超過它允許的 `VDD + 0.3V` 附近範圍。

## 34. 最後總結

如果你今天剛入門，先記這個版本就夠：

- 電池：提供能量
- 電阻：限流、分壓、拉高拉低
- 電容：穩定供電、濾波
- 電感：常在 DC-DC 和濾波裡
- 二極體：方向性與保護
- LED：會發光，要限流
- BJT：小電流控制較大電流
- MOSFET：小控制訊號控制較大負載，超常用
- 運放：整理小訊號
- 穩壓器：把電壓整理成能用的電源
- 感測器：把外界狀態變成訊號
- MCU：控制核心

最重要的一句話：

> 不要只背名字，要一直問自己：這顆元件在這個電路裡到底扮演什麼角色？

只要你開始用這個角度看電路，學得會快很多，而且比較不容易只停在背名詞。
